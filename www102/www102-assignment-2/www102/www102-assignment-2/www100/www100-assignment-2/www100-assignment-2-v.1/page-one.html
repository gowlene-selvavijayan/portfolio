Introduction

The CSS position property specifies how an element is positioned in a web document. Using CSS position, you can choose a method to visually organize web elements, the building blocks of your webpage.

The top, right, bottom, and left properties can help you move positioned elements around either on their own, or in relation to another element.

Let's learn about the five different ways you can set an element's position and adjust it using offset properties: top, right, bottom, and left.

The 5 positions values

  • static
  • relative
  • absolute
  • fixed
  • sticky

The 5 Positions Values

position: static;

When the position property is not set, the browser assumes the element is statically positioned by default. Static positioning means the element remains positioned in the order it occurs, or in other words, in the normal flow of a document.

position: relative;

Relative positioning moves the element box relative to its original position in the normal flow of a web page. In relative position, the space the element would have occupied in the normal flow is preserved and appears as visible empty space.

position: absolute;

Absolutely positioned elements are removed from the norrmal flow of the document, operating outside it entirely. These elements position themselves with respect to the viewport or their containing element.

Unlike relatively positioned elements, the space they would have occupied is closed up without affecting the positioning of surrounding elements.

An element in absolute position needs either the top or bottom properties set to appear on the page.

When you take an element out of the normal flow of a document using absolute positioning, it looks for a parent or other ancestor element with a position set (to anything other than static) because it needs an element to position itself in relation to. The furthest ancestor element a child element will position itself in relation to is the body element.

position: fixed;

What makes fixed positioning unique is that the element stays in one position in the viewport even when the document scrolls.

Fixed elements are removed from the document flow and positioned relative to the viewport rather than another element in the document.

An element in fixed positioning needs either the top or bottom properties set to appear on the page.

position: sticky;

Sticky positioning falls in between relative and fixed in that it behaves as though it is relatively positioned. However, when scrolling, once it lands on its specified position relative to the viewport, the element becomes fixed in position.

Similarly to both relative and fixed position, sticky positioning also requires a top or bottom property to function, but the element will remain visible on the page.

z-index

Arrange the layout order of elements using z-index. The higher the number set, the higher it will be in the order of elements.

For example, to ensuer a fixed or sticky navigation bar coonsistently moves on top of the rest of a web page's content, set the element's z-index to a positive integer. If no other element has its z-index set, z-index: 1; would work.
